    5. Процессы и IPC

    Подробнее по краткой лекции и плану изучения.

    Жизненный цикл процесса

    1. fork — копия родительского процесса (новый PID).
    2. exec — замена образа процесса новой программой (execve).
    3. Завершение — exit code (0 = успех, ненулевой = ошибка).
    4. Родитель читает код через wait().
    echo $?                   # exit code последней команды
sleep 5 & pid=$!
wait $pid; echo "exit: $?"

    Зомби (Zombie) — процесс завершился, но родитель не вызвал wait(). Видны как <defunct>.

    Сирота (Orphan) — родитель умер; усыновляет PID 1 (systemd).

    ps aux | awk '$8 ~ /Z/ {print}'    # зомби
ps -eo pid,ppid,stat,cmd | grep ' Z '

    Состояния процесса

    Буква в колонке STAT у ps:

    Состояние Значение
    R Running — выполняется или в очереди на CPU
    S Sleeping — ждёт события (прерываемый сон)
    D Disk sleep — ждёт I/O, нельзя убить обычным SIGKILL
    Z Zombie
    T Stopped — остановлен сигналом

    Процесс в D-state часто указывает на проблему диска или NFS.

    ps aux | awk '$8 ~ /D/ {print}'

    Сигналы

    Сигнал — асинхронное уведомление процессу.

    kill -l                   # список сигналов
kill -15 1234             # SIGTERM — вежливое завершение
kill -9 1234              # SIGKILL — немедленно, нельзя перехватить
kill -STOP 1234           # пауза
kill -CONT 1234           # продолжить
    Сигнал Номер Действие
    SIGTERM 15 Просьба завершиться (по умолчанию kill)
    SIGKILL 9 Убить безусловно
    SIGHUP 1 «Повесили терминал»; часто перечитывание конфига
    SIGINT 2 Ctrl+C
    SIGSTOP/SIGCONT 19/18 Пауза / продолжение

    Приоритеты: nice

    nice — «вежливость» к CPU: от -20 (высший приоритет) до 19 (низший). Обычный пользователь может только повышать nice (снижать приоритет).

    nice -n 10 ./heavy-job.sh
renice -n 5 -p 1234
ps -o pid,ni,cmd -p 1234

    Планировщик CFS (Completely Fair Scheduler) / EEVDF распределяет CPU-время пропорционально весу процессов.

    Потоки vs процессы

    Поток (thread) — легковесная единица внутри процесса с общей памятью. В Linux потоки — те же задачи (tasks) с общим адресным пространством.

    ps -eLf | head            # LWP = ID потока
ls /proc/$$/task/         # потоки текущей оболочки

    Файловые дескрипторы

    FD (file descriptor) — числовой идентификатор открытого файла/сокета:

    • 0 = stdin
    • 1 = stdout
    • 2 = stderr
    ls -l /proc/$$/fd/
ulimit -n                 # лимит открытых файлов (nofile)
cat /proc/1234/limits | grep "open files"

    # Перенаправление
./script.sh > out.log 2>&1
./script.sh 2>/dev/null

    Сессии, группы, демонизация

    Сессия — группа процессов с общим controlling terminal. Группа процессов — набор для управления сигналами (например, pipeline).

    # Процесс переживёт закрытие терминала
nohup ./long-task.sh &

# Новая сессия без controlling tty
setsid ./daemon.sh

    nohup игнорирует SIGHUP; setsid создаёт новую сессию.

    IPC — межпроцессное взаимодействие

    Inter-Process Communication

    Механизм Описание
    Pipe cmd1 \| cmd2 — односторонний канал
    FIFO именованный pipe (mkfifo)
    Unix-сокет сокет в файловой системе (/var/run/...)
    Shared memory общая область RAM
    Семафоры, очереди синхронизация и обмен сообщениями 
    ipcs -a                   # активные IPC-объекты
mkfifo /tmp/myfifo

    Окружение процесса

    env | grep PATH
cat /proc/1234/environ | tr '\0' '\n'
readlink /proc/1234/cwd
readlink /proc/1234/exe

    Переменные окружения (environment variables) наследуются при fork и задаются в shell, systemd-юнитах, Docker.

    Практика

    1. Запустить sleep 100 &, найти PID, отправить SIGTERM, проверить $?.
    2. Посмотреть /proc/self/fd и сопоставить с ls -l /proc/self/fd.
    3. Создать pipeline ps aux | grep ssh | wc -l и объяснить, какие процессы в одной группе.
