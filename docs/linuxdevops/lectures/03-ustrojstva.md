# 3. Устройства

Подробнее по [краткой лекции](../linux-lectures.md) и [плану изучения](../linux-plan-dim.md).

## «Всё — файл»

В Unix-подобных системах устройства представлены как специальные файлы в каталоге `/dev/`.

| Тип | Буква в `ls -l` | Пример | Назначение |
|-----|-----------------|--------|------------|
| Блочное | `b` | `/dev/sda` | Диски — чтение блоками фиксированного размера |
| Символьное | `c` | `/dev/tty`, `/dev/urandom` | Потоковый ввод-вывод |

Каждое устройство имеет **major** (тип драйвера) и **minor** (конкретный экземпляр) номера:

```bash
ls -l /dev/sda /dev/tty
# brw-rw---- 1 root disk 8, 0 ... /dev/sda   → major=8, minor=0
```

## udev

**udev** (userspace `/dev`) — демон, который реагирует на события ядра и создаёт узлы в `/dev/`, симлинки (`/dev/disk/by-uuid/`) и задаёт права.

Правила лежат в `/etc/udev/rules.d/`:

```bash
# Информация об устройстве
udevadm info -a -n /dev/sda | less

# Следить за событиями в реальном времени
sudo udevadm monitor

# Переприменить правила (после подключения USB и т.п.)
sudo udevadm trigger
```

## sysfs

**sysfs** — виртуальная ФС в `/sys/`: дерево устройств, драйверов, параметров питания, скорости вентиляторов. Дополняет `/proc` (процессы) информацией об **железе**.

```bash
# Драйвер сетевой карты
ls -l /sys/class/net/
cat /sys/class/net/eth0/operstate
```

## Инвентаризация железа

| Команда | Что показывает |
|---------|----------------|
| `lspci` | Устройства на шине PCI (диски, сеть, GPU) |
| `lsusb` | USB-устройства |
| `lsblk` | Блочные устройства и разделы (дерево) |
| `lscpu` | CPU: ядра, архитектура, флаги |
| `lshw` | Подробный отчёт (нужен root для всего) |
| `dmidecode` | DMI/SMBIOS — модель сервера, слоты RAM |

```bash
lspci -k                  # + какой драйвер (kernel driver) привязан
lsblk -f                  # ФС, UUID, LABEL
lscpu
sudo dmidecode -t system    # производитель и модель
```

**Драйвер** устройства в Linux — это обычно **модуль ядра** или встроенный код ядра.

## Псевдоустройства

| Устройство | Назначение |
|------------|------------|
| `/dev/null` | Поглощает любой ввод («чёрная дыра») |
| `/dev/zero` | Бесконечный поток нулевых байт |
| `/dev/urandom` | Криптографически стойкие случайные байты |
| `/dev/random` | Блокируется, если мало энтропии |

```bash
# Записать 100 МБ нулей в файл
dd if=/dev/zero of=/tmp/test.img bs=1M count=100

# Loop-устройство: файл как диск
sudo losetup -f /tmp/test.img
losetup -a
sudo losetup -d /dev/loop0   # отключить
```

**Loop device** — способ смонтировать образ диска (ISO, контейнерный слой) как блочное устройство.

## Терминалы: tty и pts (обзорно)

- **tty** (teletype) — физические или виртуальные консоли (`Ctrl+Alt+F1`…).
- **pts** (pseudo-terminal slave) — псевдотерминалы для SSH-сессий и терминальных эмуляторов.

```bash
tty                       # ваш текущий терминал
who                       # кто залогинен и откуда
ls /dev/pts/
```

## Практика

1. `lsblk -f` — сопоставить разделы, UUID и точки монтирования.
2. `lspci -k` — найти сетевую карту и её драйвер.
3. `udevadm info` для USB-флешки после подключения.
