# 2. Ядро

Подробнее по [краткой лекции](../linux-lectures.md) и [плану изучения](../linux-plan-dim.md).

## Версии и обновление

**Ядро (kernel)** — центральная часть Linux. Его версия не совпадает с версией дистрибутива.

```bash
uname -r                    # версия ядра
uname -a                    # полная информация
rpm -q kernel               # установленные пакеты ядра (RHEL)
dpkg -l 'linux-image*'      # то же на Debian/Ubuntu
```

После установки нового ядра нужна **перезагрузка**. Старое ядро обычно остаётся в GRUB — это запасной вариант при сбое.

## Модули ядра

**Модуль** — подгружаемый драйвер или расширение ядра. Не всё вшито в образ ядра: часть загружается по требованию.

```bash
lsmod                       # загруженные модули
modinfo e1000               # информация о модуле (зависимости, параметры)
sudo modprobe br_netfilter  # загрузить модуль
sudo modprobe -r br_netfilter  # выгрузить

# Автозагрузка при старте
ls /etc/modules-load.d/
echo 'br_netfilter' | sudo tee /etc/modules-load.d/br_netfilter.conf

# Blacklist — запретить загрузку
echo 'blacklist nouveau' | sudo tee /etc/modprobe.d/blacklist-nouveau.conf
```

## Параметры ядра: sysctl

Параметры ядра доступны как файлы в `/proc/sys/`. Утилита **sysctl** читает и меняет их.

```bash
# Текущее значение
sysctl vm.swappiness
cat /proc/sys/vm/swappiness

# Изменить до перезагрузки
sudo sysctl -w vm.swappiness=10

# Сохранить навсегда
echo 'vm.swappiness=10' | sudo tee /etc/sysctl.d/99-swappiness.conf
sudo sysctl --system    # применить все файлы из /etc/sysctl.d/
```

**swappiness** — насколько агрессивно ядро сбрасывает страницы памяти в **swap** (файл/раздел подкачки). 0 — почти не использовать swap, 100 — активно.

## Сообщения ядра

Ядро пишет диагностику в **кольцевой буфер** (ring buffer) — старые сообщения затираются новыми.

```bash
dmesg -T                  # с человекочитаемым временем
dmesg -T | tail -30
dmesg -w                  # следить в реальном времени
journalctl -k             # только kernel, через journald
journalctl -k -p err      # ошибки ядра
```

## Kernel panic, SysRq, kdump (обзорно)

**Kernel panic** — критическая ошибка, после которой ядро останавливает систему (аналог «синего экрана»).

**Magic SysRq** — аварийные команды ядру через комбинации клавиш (на серверах часто отключено). Пример: синхронизация дисков перед жёсткой перезагрузкой.

**kdump** — механизм сохранения дампа памяти при panic для последующего анализа. Настраивается отдельно, на экзамене/в проде достаточно знать, что это существует.

```bash
# Включён ли SysRq (1 = все команды разрешены)
cat /proc/sys/kernel/sysrq
```

## OOM killer

**OOM** (Out Of Memory) — ситуация, когда RAM и swap исчерпаны. Ядро выбирает процесс-жертву и убивает его сигналом SIGKILL.

```bash
# Оценка «жертвенности» процесса (чем выше — тем вероятнее убийство)
cat /proc/1234/oom_score

# Корректировка (-1000 = почти никогда не убивать, 1000 = первый кандидат)
echo -100 | sudo tee /proc/1234/oom_score_adj

# Логи OOM
dmesg -T | grep -i 'out of memory'
journalctl -k | grep -i oom
```

## Виртуальная память

Память процесса разбита на **страницы** (pages), обычно по 4 КБ. Неактивные страницы могут уйти в **swap**.

**Page cache** — кэш содержимого файлов в RAM. Поэтому `free -h` показывает мало «свободной» памяти, хотя система не перегружена: кэш отдаётся приложениям при необходимости.

```bash
free -h
cat /proc/meminfo | head -20
```

## eBPF (обзорно)

**eBPF** (extended Berkeley Packet Filter) — механизм запуска небольших проверенных программ **внутри ядра** без загрузки модулей. Используется для:

- мониторинга (bcc-tools, bpftrace);
- сетевой фильтрации (Cilium);
- безопасности (falco).

Для DevOps достаточно понимать: eBPF даёт наблюдаемость с минимальным overhead, это современная альтернатива многим «костылям» на `/proc`.

```bash
# Проверить поддержку (если установлен bpftool)
bpftool feature probe 2>/dev/null | head
```
